# Evidence Patterns

Use concrete values in acceptance progress notes.

Good:

- `exit=0, 18 passed`
- `GET /health -> 200, body.status=ok`
- `created dist/index.html, size=14KB`
- `button text='Save', toast='Saved'`

Bad:

- `looks implemented`
- `should work`
- `covered by previous test`
- `same as AT-02`
