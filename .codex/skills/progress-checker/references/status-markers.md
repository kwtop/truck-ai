# Status Markers

Use the same markers across all automation skills:

| Marker | Meaning |
|--------|---------|
| `[ ]` | Not started |
| `[~]` | In progress |
| `[x]` | Completed |
| `[!]` | Blocked |

`auto-coder` should only write `[x]` after the task has been implemented and verified.
`progress-checker` may report `[!]` if a human decision or credential is required.
