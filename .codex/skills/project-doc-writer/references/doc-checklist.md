# Documentation Checklist

Use this checklist when reviewing generated docs:

- Purpose is clear in the first paragraph.
- Features match `DEV_SPEC.md`.
- Setup commands match actual package files.
- Run and test commands are present.
- Required environment variables are listed with placeholders.
- Architecture notes describe real directories/modules.
- Known gaps and unverified assumptions are explicit.
- No real secrets or local machine paths are exposed.
